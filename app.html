POST /login¶

Authenticates the user. If no user with the provided username exists or if the password is incorrect redirect to the login page and flash a message notifying the user of the problem. Otherwise redirect to the users home page.

Form Parameters:
 
  • username – The username the user is trying to log in with.
  • password – The password the user is trying to log in with.
GET /chat¶

Shows a web chat with chat histories between the current user and his contacts. If the current user is a member of the nursing care group all users will be his contacts. Otherwise only users in the nursing care group will be his contacts.

GET /¶

Shows the users home page, listing documents which were changed since their last access as well as unread messages. Only documents related to patients assigned to the current user and messages by the current user’s contacts are listed.

Query Parameters:
 
  • filter-by – One of patient, sender, timestamp.
  • id – The id of the patient or sender if filter-by is patient or sender. Only unread documents and messages by this patient or sender are listed.
  • start-date – The date of the earliest documents and messages to be shown. Defaults to the earliest representable date.
  • start-time – The earliest time used in conjunction with start-date. Defaults to 00:00.
  • end-date – The date of the newest documents and messages to be shown. Defaults to the current date.
  • end-date – The latest time used in conjunction with end-date. Defaults to 23:59.

GET /patients/(patient_id)/calendar/schedules/(schedule_id)/(action)¶

Accepts or declines an appointment suggestion. A declined appointment is deleted.

Parameters:
  • patient_id (str) – The id of the patient whose schedule entry is to be accepted or declined.
  • schedule_id (str) – The id of the schedule entry to be accepted or declined.
  • action (str) – The action to be taken. Accepted values are accept and decline. If a different action is specified no changes will be made.

Translations¶

This module provides translations and a helper function intended to be used as a Jinja template filter.

app.translations.translations = {}¶

Used to specify translations. If the value is a tuple the first value specifies the singular translation, the second value the plural translation.

Type:dict[str, str] or dict[str, tuple(str, str)]
Example: 
>>> translations = {'singular': 'singular'}
>>> translations = {'both': ('singular', 'plural')}
app.translations.translate(token, plural=False)¶

Returns the translation if one exists, otherwise return the original token.

Parameters:
  • token (str) – A str token used to find the translation.
  • plural (bool) – Specifies whether the plural translation should be returned, if one exists.

The translation, or token.

Utilities¶

This module provides various utility functions.

app.utilities.incremental_ids()¶

Returns a generator which yields an incrementing alphabetic identifier, starting at ‘A’, prepending another letter to prevent overflow if necessary.

Returns:A generator which yields the next identifier.
Return type:Generator[str]
Example: 
>>> g = incremental_ids()
>>> next(g)
'A'
>>> next(g)
'B'
>>> while next(g) != 'Z':
...     pass
>>> next(g)
'AA'

app.utilities.iso8601_to_utc_datetime(s)¶

Converts an ISO 8601 formatted str in Zulu time without microseconds to its respective datetime object.

Parameters:s (str) – An ISO 8601 formatted str in Zulu time without microseconds.
Returns:A datetime object representing the provided time with the timezone set to UTC.
Return type:datetime.datetime
Example: 
>>> iso8601_to_utc_datetime('1970-01-01T00:00:00Z')
datetime.datetime(1970, 1, 1, 0, 0, tzinfo=datetime.timezone.utc)
